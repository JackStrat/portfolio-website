evrySavings & Donations | Responsive Web / iOS App

evry little savings.

A savings platform enabling every individual to save & donate money, in a transparent and simplistic way.

The objective was to help people save money quickly in preparation for a big purchase or expenditure. Build trust and confidence. Create and design a cohesive brand, iOS and responsive web app.

Process

Process Steps

Firstly, a set of brand guidelines was created based on a partially given design brief. Consolidated into a brand guidelines document with an accompanying client proposal.

Next, user flows were created from user stories, to map the path taken by a prototypical user to complete a task.

These flows were transformed into low, mid to high-fidelity wireframes of key screen layouts before testing on users to discover potential pain points.

The user testing results led to improvements and iterations for the final designs.

User Flows

User flows were created from user stories and key functionalities, to map the path taken by a prototypical user to complete a task.

User Flow Diagram

User flows were translated to a diagram to figure out the flow of the app after having thought about the customer experience and user needs. To best understand the needs and the experience of the user, it was important to map and visualize them.

Low Fidelity Wireframes

Low fidelity wireframes were created from the user flows and diagram, enabling to have the bare-bones elements that were to be included in the finished project. Sketches were relatively detailed to help with content creation further down the line.

Mid Fidelity Wireframes

Medium fidelity wireframes were created for user flow 3 as a step up from low fidelity, with more detail including accurate spacing, headlines, and buttons.

These mid-fidelity wireframes were transformed into a prototype in preparation for the next stage in the process, a scenario based user test.

High Fidelity Wireframes

After conducting the scenario based user test, compiling and analysing research notes, all designs were revised and pain points resolved. Producing a set of high fidelity wireframes, including typography, illustrations, icons, and CTA buttons.

Colour was addressed and applied later in the final User Interface design screens, referencing the Brand guidelines meticulously.

Responsive Web App Designs

Responsive web app screens were designed to accommodate different screen sizes, so that the content looks great on any screen size.

The following responsive web designs were created using the “mobile-first” principle, whereby the experience was defined first on mobile and then scaled up to larger screens (also known as “breakpoints”) such as a tablet, desktop, and desktop HD.

Conclusion

The mission was to create a product for every individual to save money, in a transparent and simplistic way in preparation for a big purchase or expenditure.

The unique selling point that the community can choose to help charities as they save, builds trust and confidence in the company.

Personality is reflected through the colour palette, white for clarity and predominantly deep blues communicating a feeling of professionalism and reliability. This can also be said for the illustrative style, as well as giving a friendly and human touch to the brand.

It’s understood that saving can tend to be complicated, to counteract this feeling, the typographic set is contemporary, approachable and clear to read.

The users don’t need to be a financial expert to save, that’s why it was kept simple.

Every little goes a long way.
